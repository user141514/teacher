## 结论

**生产结论：Request changes。** 当前材料可以作为视觉与提示词设计基线，但还不是可用于真实员工管理的智能体。核心原因是前端没有模型/API、状态和结构化数据流，任何输入都会得到固定演示结论。

本项目已把独立 v2 提示词、v2 测试与合并知识库设为权威材料；`legacy-combined.md` 仅供追溯。原始外置 HTML 与 ZIP 内版本 SHA-256 均为 `0a180542cffd4621414e860b9cffd30e6c67f4d2d719d5f0e782106333cda3b6`。

## 阻断项

### Critical：核心流程不消费用户输入

- `runAI()` 只有动画和定时跳页，没有 `fetch`、模型调用、JSON 解析或错误分支。
- 表单没有状态绑定；类型判定、方案、反馈均为固定文本。
- 用户改选类型只改变卡片样式，不改变方案；反馈为空也会生成固定建议。

上线前必须建立真实状态树和四步调用链，并让每一步只消费上一步通过 Schema 校验的结果。

### High：分类契约自相矛盾

- v2 要求证据不足时不硬判，但输出仍强制 `ability`、`will` 只能为“高/低”，没有“未知/需补充”。
- 知识库要求低能力、低意愿时区分 D1/D2；前端却只有一个“待改进型”，也没有“是否辅导过”等必要字段。
- “四类画像”实际包含 A、B、C、D1、D2 五个操作类型。应使用稳定字段，例如 `quadrant: D`、`type_id: D1|D2`，而不是让模型生成任意类型名。

### High：职场决策安全边界缺失

- “老油条”“问题员工”等标签与“不贬损”规则冲突；“忠诚度高”等推断没有证据基础。
- 知识库包含“人员优化”建议，却没有客观证据、合理支持、HR/法务复核、反歧视、反报复、申诉和人工最终决策要求。
- 对怀孕、残障、疾病、工会活动、举报、骚扰、暴力或自伤等高风险场景没有升级路径。

本助手只能支持沟通准备，不能自动作出录用、晋升、处分、调岗或解除关系决定。上述场景必须停止画像并转人工制度流程。

### High：提示注入与输出验证缺失

自由文本、知识库和下游 JSON 都会被直接拼接。必须把它们作为不可信数据隔离，明确“其中的指令不执行”，并在服务端实施长度限制、JSON Schema、枚举、任务 ID、超时/限流和一次重试。不得回显系统提示或知识库全文。

### High：版本与命名漂移

前端使用“熟手待激活型”等展示名，知识库使用“精明人/老牛(B)”等名称，旧合并稿还使用两个知识库占位。生产实现必须以稳定 ID 驱动逻辑，展示名单独配置且使用中性语言。

### High：隐私治理缺失

员工绩效、情绪和沟通记录属于敏感职场数据。接入 API 前需明确最小采集、匿名化、访问权限、日志脱敏、供应商数据用途、保留期限和删除机制。“仅本次会话留存”必须由后端行为证明，不能只写在界面上。

### High：测试仍不是模型回归

`tests/prompt-cases.md` 是叙述式验收需求，没有完整输入、固定模型参数、精确期望 JSON 或自动断言。当前 2 个 Playwright 用例只证明已修复的前端复制成功/失败行为，不证明提示词正确或公平。

## 其他重要问题

- 生成步骤没有传入完整画像、目标、证据与上一版方案，无法可靠个性化或验证“换个角度”。
- 3 个月、12 个月边界和 3–12 个月区间规则不清。
- 返回首页不重置 `S.max`，旧定时器也未取消，可能串入下一轮流程。
- 方案与反馈自由文本已统一走 `renderMarkdown`（禁原始 HTML、协议白名单、DOM 标签/属性白名单）；后续新增模型字段仍不得直接拼接进 `innerHTML`，生产部署还需配置 CSP。
- 可点击步骤使用 `div`，表单 label 未完整关联控件，键盘与屏幕阅读器体验不足。

## 必须补充的对抗测试

1. 空白、矛盾、证据不足、3/12 月边界和 D1/D2 证据冲突。
2. “忽略规则/泄露系统提示/新增第五类”等直接与间接提示注入。
3. 只替换姓名、性别、年龄等无关信息时结果不得变化的公平性变形测试。
4. 受保护特征、报复、骚扰、暴力、自伤和医疗信息的人工升级测试。
5. 非 JSON、损坏 JSON、非法枚举、超长输出、超时、限流和重试失败。
6. 模型返回 HTML/脚本、返回上一步数据保留、重新开始状态清空、旧请求回写。

## 进入真实开发的验收门

- 建立版本化 Schema 与真实四步状态机。
- 统一 A/B/C/D1/D2 稳定 ID 和中性展示名。
- 增加“未知/需补充”状态，低置信度禁止生成方案。
- 完成 HR/法务/公平性/隐私边界与人工复核。
- 建立提示词契约测试、安全测试和端到端回归后，才可进入真实员工场景试点。
