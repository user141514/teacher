# 教练助手前台常驻启动脚本设计

## 背景

当前 `start.bat` 调用 `scripts/start.ps1`，PowerShell 使用隐藏的后台 Node 进程启动服务，健康检查通过后立即退出。因此双击启动后，终端不会持续承载服务；关闭原终端也不会终止已经脱离的 Node 进程，`4173` 端口可能继续被占用。

目标行为参考 `D:\codex-pj\time\start.bat`：服务进程与启动终端保持同一生命周期，终端在服务运行期间持续显示日志，关闭终端或按 `Ctrl+C` 后，本次启动的服务进程结束并释放端口。

## 目标

- `start.bat` 启动后终端保持打开并显示 Node 服务日志。
- 正常退出、`Ctrl+C` 或关闭终端时，只终止本次启动的 Node 进程，端口随之释放。
- 启动前若配置端口已经被监听，安全提示并退出；不得复用、接管或终止已有进程。
- 保留 Node.js、`.env`、`node_modules` 检查，以及 `-CheckOnly`、`-NoBrowser` 参数。
- 浏览器只在本次服务通过健康检查后打开。
- 不读取、打印或复制 `.env` 中的密钥，只读取 `PORT`；未配置时使用服务默认端口 `4173`。

## 非目标

- 不增加 Windows 服务、守护进程或进程管理器。
- 不实现强制清理任意端口占用进程。
- 不修改服务端 API、DeepSeek 配置、前端代码或依赖。
- 不改变 `.env` 和 `node_modules` 的安装/创建方式。

## 方案

### 启动入口

继续保留 `start.bat → scripts/start.ps1` 的单一实现路径。`start.bat` 负责切换到项目目录、转发参数和在启动失败时保留错误信息；环境检查、端口检查、浏览器打开及 Node 生命周期管理继续集中在 `scripts/start.ps1`，避免在批处理和 PowerShell 中复制两套逻辑。

### 端口解析与安全检查

PowerShell 在确认 Node.js 和 `.env` 存在后，通过 Node 的 `--env-file=.env` 仅求值 `process.env.PORT ?? 4173`。结果必须是 `1` 至 `65535` 的整数，否则返回固定配置错误。

启动 Node 前读取本机活动 TCP 监听端点。只要目标端口已经被监听，就输出“端口已被占用，未启动服务”的安全提示并返回非零状态。脚本不得根据端口反查并终止 PID，也不再复用现有健康服务。

### 前台进程生命周期

PowerShell 使用 `Start-Process` 的 `-NoNewWindow -PassThru` 启动 Node，使其日志进入当前终端，同时保存本次 Node PID。脚本完成健康检查和可选的浏览器打开后，等待该 Node 进程退出，因此终端始终被占用。

Node 进程管理放在 `try/finally` 中：正常退出时返回 Node 的退出码；脚本被 `Ctrl+C` 中断且 Node 仍存活时，只对保存的本次 PID 执行 `Stop-Process`。关闭整个终端窗口时，Windows 会结束同一控制台进程树，Node 释放其监听端口。脚本永远不以“端口号”为依据杀进程，从而避免误伤后来占用同一端口的其他程序。

### 健康检查与浏览器

Node 启动后最多等待 20 秒访问动态端口对应的 `/api/health`。健康后输出服务地址，并在未指定 `-NoBrowser` 时打开浏览器；随后继续等待 Node 退出。若 Node 提前退出或超时未健康，返回固定启动失败提示，并由 `finally` 清理仍存活的本次进程。

`-CheckOnly` 只验证 Node.js、npm、`.env`、`node_modules`、端口配置和端口可用性，不启动 Node 或浏览器。

## 测试设计

在 `tests/start-script.test.js` 中沿用临时工作区，不调用真实模型 API：

1. 保留缺少 `.env`、缺少依赖、`-CheckOnly` 和批处理参数转发测试。
2. 使用临时端口和最小本地 HTTP fixture 启动 `start.bat -NoBrowser`，确认健康后批处理进程仍未退出，证明服务在前台常驻。
3. 终止该测试进程树后轮询端口，确认端口释放。
4. 预先用 fixture 占用目标端口，再运行启动脚本；断言脚本非零退出、原 fixture 仍健康，证明安全模式不杀已有进程。
5. 所有 fixture 仅返回本地 `/api/health`，不读取真实 `.env` 或发送外部请求。

## 文档与兼容性

同步更新 `README.md` 的本地启动说明：删除“后台启动并复用现有服务”的旧描述，说明终端必须保持打开、关闭终端会停止本次服务、端口已占用时安全退出。

既有 `start.bat` 双击入口、`scripts/start.ps1 -NoBrowser` 和 `scripts/start.ps1 -CheckOnly` 调用方式保持不变。

## 验收标准

- 启动成功后终端持续占用并显示服务日志。
- 浏览器在健康检查成功后打开；`-NoBrowser` 不打开浏览器。
- 关闭启动终端或按 `Ctrl+C` 后，本次 Node 进程不再存活，配置端口可重新绑定。
- 端口已被其他进程占用时，脚本安全退出且原进程保持运行。
- 聚焦启动脚本测试和项目全量测试通过。
- 不修改或提交用户现有的前端测试、Playwright 临时配置、历史文档或 `output/`。
