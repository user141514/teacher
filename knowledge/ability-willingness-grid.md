# 教练助手 · 知识库(能力×意愿 合并网格)

> 用途:作为「教练助手」类型判定与方案生成两步的**唯一事实源**,以 `{{画像知识库}}` 整块注入提示词。
> 说明:原素材有两张图——「人才四象限(意愿×能力)」管**用人**、「因材施教(动力×功力)」管**教练法**。二者坐标一致(动力=意愿、功力=能力),此处合并为一张网格,避免两步归类不一致。
> 维护:本文件是单一事实源;提示词只引用、不复制,更新画像/策略只改这里。

## 一、维度判定线索(先据此判"高/低",再查表)

**能力(功力)——看能否稳定、独立地把事做成**
- 高:经验足/在岗较久,绩效稳定达标,能独立交付,技能熟练。
- 低:经验浅或频繁出错,需带教,难独立完成,绩效不稳。

**意愿(动力)——看主动性、投入度与情绪状态**
- 高:主动接挑战、投入,愿意成长,情绪积极。
- 低:被动/推脱/讨价还价,只守舒适区,或情绪抵触、消极怠工。

> 判定要求:能力、意愿各须引用输入中的**具体证据**;任一维证据不足以定高低时,可标为“未知”，置信度记“低”并回上一步追问。能力或意愿为“未知”时，status="待补充"，不得输出象限或类型；证据相互冲突时，status="待人工确认"，不得输出类型。

## 二、四格主表(每格挂:用人策略 / 教练模式 / 常用技术 / 沟通与话术要点 / 建议频率)

### 格 A|能力足 · 意愿足 —— 明星员工 / 优秀资深(教练法:授权式)
- **用人策略**:委以重任,授权拔高。
- **教练模式**:提供分享机会、梳理其经验;赋予导师角色实现自我价值;充分授权、激发潜力;以更高挑战强化实力。
- **常用技术**:高度信任 Trust(充分授权、给发挥潜力的机会,满足自尊)+ 开放探索 Open(提有挑战性的问题,不轻易给意见,由对方自主决策)。
- **沟通与话术要点**:匹配与贡献相应的薪酬回报;关注工作负荷,避免"能者过劳";保持任务多样性,防职业倦怠与流失。
- **建议频率**:低频、结果导向,高授权。

### 格 B|能力足 · 意愿不足 —— 成熟待激活型(教练法:诱导式)
- **用人策略**:激发意愿。
- **教练模式**:融入其境、争取信任;追溯阻碍其发展动力的因素;探索内心多元需求;设计诱因、激发新动力;以外在刺激驱动执行力。
- **常用技术**:建立信任 Trust(不贴标签、保持尊重,感同身受争取信任)+ 开放探索 Open(中性开放提问,引导其自诉障碍、需求与期望)。
- **沟通与话术要点**:了解其真实工作动机,找到对应激励或合适任务;坦率沟通"付出与收获成正比",点明只守舒适区无法获得更好回报与发展。
- **建议频率**:中频,需持续督促与提醒。

### 格 C|能力不足 · 意愿足 —— 潜力股 / 充满激情的新手(教练法:引导式)
- **用人策略**:长期培养。
- **教练模式**:激励其分享想法、发掘天分;不要太快给提示,更多启动其思考;给更多尝试机会加速学习;引导自掘、自寻答案;要求提炼总结并持续修炼。
- **常用技术**:开放探索 Open(开放提问、鼓励分享)+ 分析问题 Problem(以提问逻辑层层剖析,训练洞察根因)+ 共谋对策 Solution(给思考与尝试机会,加速学习解决问题)。
- **沟通与话术要点**:以其可胜任的工作为主,辅以挑战性任务循序渐进;做好辅导准备、给予试错机会;这类人忠诚度高、可培养为左膀右臂。
- **建议频率**:较高频,带教 + 及时反馈。

### 格 D|能力不足 · 意愿不足 —— ⚠ 需二次区分(见第三节)
此格对应两类**完全不同**的人,必须借"入职时长 + 绩效历史"区分后再给策略:
- **D1 新入职适应型**——教练法:**教导式**
  - 用人/教练:给清晰目标并确保理解;工作简单化、标准化、易出成效;明确指示、实操性指导;短间距跟进与支持;有成效及时肯定表扬;积累成功经验、逐步强化信心。
  - 常用技术:建立信任 Trust(以帮助对方成功的姿态提要求)+ 共识行动 Action(清晰沟通、确保行动一致)。
  - 频率:高频、短间距。
- **D2 绩效改进支持型(长期未达标且已多次辅导的员工)**——策略:**绩效改进支持**
  - 先进行绩效面谈,提出明确改进要求。
  - 若辅导后仍消极怠工、散播负面情绪破坏团队，转人工按公司人事流程评估；教练助手停止生成处置建议。
  - 若愿意改进:日常主要安排辅助性工作,并结合绩效改善目标布置任务,针对性补齐岗位所需能力。
  - 频率:改进期高频跟踪 + 明确里程碑。

## 三、左下格区分规则(关键,判定必须执行)

当 ability=低 且 will=低 时,按以下优先级判 **D1 还是 D2**:
1. **入职时长**:3 个月内(新人)→ 倾向 D1;1 年以上 → 倾向 D2；入职满 3 至 12 个月时，必须补齐绩效周期与辅导历史，不能仅以时长判定。
2. **绩效历史**:尚无历史/首个周期 → 倾向 D1;长期/连续未达标 → 倾向 D2。
3. **是否已辅导过**:未辅导过、给机会即改善 → 倾向 D1;已多次辅导仍无改善 → 倾向 D2。

> 若上述线索缺失,不得在 D1/D2 间硬选:置信度记"低"、status="待补充"、quadrant/type_id 均为 null，并回上一步追问"入职时长 / 绩效历史 / 绩效周期 / 辅导历史"。若线索互相矛盾，status="待人工确认"，转人工澄清。这也是信息输入表单必须收集这些字段的原因。

> 高风险人事处置边界:当输入涉及解雇、辞退、处分、降级或其他需人事/法务判断的处置时，教练助手不得生成教练或处置方案，必须停止生成并转人工处理。
